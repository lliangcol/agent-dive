# Roadmap

AgentDive 按“先沉淀高质量学习资产，再逐步自动化”的原则推进。

## Phase 1：骨架与规范

- [x] 明确项目定位、目录结构和收录边界
- [x] 建立项目索引、学习进度、贡献指南和执行说明
- [x] 建立项目分析、图解、学习任务和收录报告模板
- [x] 建立知识体系和阶段学习路线

## Phase 2：首批人工收录样例

- [x] 选择首批代表性 AI Agent 项目
- [x] 每个项目使用 `owner__repo` 作为隔离 ID
- [x] 为每个样例生成项目精读、图解和 Learning Todo List
- [ ] 记录可复核的运行、测试和源码验证证据
- [ ] 完成首轮模板质量校准

## Phase 2.5：质量治理和学习闭环

- [x] 统一项目 `meta.json` 和学习 `progress.json` 数据契约
- [x] 为每个项目建立 `evidence.md`
- [x] 将 Learning Todo List 改为可执行、可验收、可由 Agent 判断的任务表
- [x] 建立 AI Agent 连续学习协议
- [x] 增加轻量一致性检查脚本
- [ ] 完成 CodeGraph evidence-backed 金样本

## Phase 3：半自动化收录

- [x] 提供轻量一致性检查脚本
- [ ] 提供轻量收录脚本或命令入口
- [ ] 自动创建项目目录和学习笔记目录
- [ ] 自动更新 `PROJECTS.md` 与 `LEARNING_PROGRESS.md`
- [ ] 自动生成基础 Mermaid 图解草稿
- [ ] 增加收录报告和质量门禁检查

## Phase 4：学习监督

- [ ] 根据 Learning Todo List 检查学习进度
- [ ] 生成复习问题和阶段总结建议
- [ ] 对学习笔记做结构化 Review
- [ ] 识别卡点并推荐下一步任务

## Phase 5：开源协作与知识平台

- [ ] 完善贡献流程和评审标准
- [ ] 建立跨项目对比和选型指南
- [ ] 沉淀生产可用性、安全、评测和可观测性清单
- [ ] 评估是否需要文档站或轻量检索能力

## 主分支

本项目公开文档、贡献说明和后续提交默认以 `main` 作为主分支名称。
