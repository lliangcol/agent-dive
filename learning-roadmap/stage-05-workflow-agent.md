# Stage 05：Workflow Agent

## 阶段目标

理解用明确工作流约束 Agent 行为的方法，掌握节点、边、状态和失败路径。

## 必备知识

- DAG 或状态图。
- Planner / Executor 模式。
- 任务编排和状态传递。

## 推荐任务

- 找到项目中的 workflow 定义。
- 画出一个典型任务的节点流转。
- 记录节点失败、重试和回滚策略。

## 实践产出

- Workflow 图。
- 一个节点职责表。

## 检查问题

- 哪些步骤由模型决定，哪些步骤由工作流固定？
- 状态如何在节点间传递？
- 工作流如何处理异常？
