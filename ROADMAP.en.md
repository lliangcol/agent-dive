# Roadmap

AgentDive is promoted based on the principle of “precipitating high-quality learning assets first, and then gradually automating them”.

## Phase 1: Skeleton and Specifications

- [x] Clarify project location, directory structure and collect boundaries
- [x] Establish project index, learning progress, contribution guide and execution instructions
- [x] Create project analysis, diagrams, learning tasks and collect report templates
- [x] Establish knowledge system and staged learning route

## Phase 2: The first batch of manual collection samples

- [x] Select the first batch of representative AI Agent projects
- [x] Use `owner__repo` as quarantine ID per project
- [x] Generate project deep dive, diagrams and Learning Todo List for each sample
- [ ] Record reviewable running, testing and source code verification evidence
- [ ] completed the first round of template quality calibration

## Phase 2.5: Quality governance and learning closed loop

- [x] Unify Project `meta.json` and learn `progress.json` Data Contracts
- [x] Create `evidence.md` for each project
- [x] Change the Learning Todo List to a task list that is executable, acceptable, and judged by the Agent
- [x] Establish AI Agent continuous learning protocol
- [x] Add lightweight consistency check script
- [ ] Completion of CodeGraph evidence-backed gold samples

## Phase 3: Semi-automated collect

- [x] Provide lightweight consistency check script
- [ ] Provides lightweight collect script or command entry
- [ ] Automatically create project directory and learning notes directory
- [ ] Automatically update `PROJECTS.md` and `LEARNING_PROGRESS.md`
- [ ] Automatically generate drafts of basic Mermaid diagrams
- [ ] Add collect report and quality access control check

## Phase 4: Learning Supervision

- [ ] Check learning progress according to Learning Todo List
- [ ] Generate review questions and stage summary suggestions
- [ ] Conduct structured review of learning notes
- [ ] Identify stuck points and recommend next tasks

## Phase 5: Open Source Collaboration and Knowledge Platform

- [ ] Improve the contribution process and review standards
- [ ] Establish cross-project comparison and selection guide
- [ ] Precipitation production availability, safety, measurement and observability checklist
- [ ] Evaluate whether you need a documentation station or lightweight retrieval capabilities

## Master branch

This project’s public documents, contribution instructions, and subsequent submissions use `main` as the main branch name by default.
