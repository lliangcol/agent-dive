# Caveman Study Plan

## Learning Objectives

- Understand how Caveman distributes output compression rules to multiple AI coding assistants.
- Run through a dry-run without writing, and record the provider matrix.
- Understand Claude Code hooks, statusline, mode flag and auto-activation paths.
- Source code confirms the key call chains of installer, MCP shrink, and memory compression.
- Summarize its reference value for Agentic Coding, token governance and skill distribution.

## Stage arrangement

1. Understand the project: read the README, INSTALL.md, CLAUDE.md and hooks / MCP documentation.
2. Run-through use: dry-run first, and then install only one Agent path in the isolation environment.
3. Source reading: Enter from `bin/install.js`, `src/hooks/`, `src/mcp-servers/caveman-shrink/`.
4. Integration verification: Test Codex single path, MCP shrink and non-sensitive compression memory.
5. Summary evaluation: Compare the optimization of ordinary concise prompt, editor rule files, Codex skill and MCP metadata.

## Validate boundaries

- Do not write the installation write range as a fact before dry-run is completed.
- Do not write the README / INSTALL.md process as a real call chain before reading the source code.
- Sensitive memory files will not be executed `caveman-compress` before API/CLI fallback is confirmed.
- Before the benchmark is reproduced, README token savings will not be regarded as local workflow benefits.
