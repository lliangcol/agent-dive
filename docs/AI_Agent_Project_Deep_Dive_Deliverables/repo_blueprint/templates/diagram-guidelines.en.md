# Diagram Guidelines

## Required Diagrams

- architecture.mmd
- flow.mmd
- sequence.mmd

## Optional Diagrams

- integration.mmd
- troubleshooting.mmd
- agent-loop.mmd
- rag-flow.mmd
- mcp-tooling.mmd

## Diagram Requirements

- Each diagram expresses only one theme.
- Module naming is consistent with the text.
- Arrow direction must be accurate.
- Speculative content must be marked.
- Each diagram must include an explanatory note below it.
- Diagrams for the same project should use a consistent style.
