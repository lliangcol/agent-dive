# Context Engineering

## 定义

Context Engineering 是围绕模型上下文窗口组织信息的工程方法，包括选择、排序、压缩、裁剪、引用和注入。

## 在 AI Agent 项目中的作用

Agent 的质量很大程度取决于上下文是否包含正确材料、约束和任务状态。项目精读、代码分析和工具调用都需要稳定的上下文组织策略。

## 典型实现方式

- 系统提示、开发者提示和用户输入分层。
- RAG 检索结果按相关性和来源组织。
- 长对话摘要和任务状态压缩。
- 重要约束前置，低价值历史裁剪。

## 常见误区

- 把所有材料都塞进上下文。
- 关键约束位置太靠后。
- 混合事实、推测和用户偏好。
- 没有记录上下文来自哪里。

## 推荐学习任务

- 记录项目构造 prompt 的入口。
- 分析上下文包含哪些材料。
- 找到裁剪、摘要或压缩逻辑。
- 用一个失败案例检查上下文缺失原因。
