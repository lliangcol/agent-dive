# Multi-Agent

## Definition

Multi-Agent refers to a system form in which multiple agents work together to complete tasks through division of labor, collaboration, debate, review, or assembly lines.

## Role in the AI Agent project

Multi-Agent can split complex tasks into roles such as planning, execution, review, testing, and summary, but it will also introduce problems of cost, delay, conflict, and state consistency.

## Typical implementation

- Supervisor assigns tasks to multiple Workers.
- Planner, Executor, Reviewer hierarchical collaboration.
- Multi-Agent conversations or debates.
- Role node execution based on workflow graph.

## Common misunderstandings

- Excessive splitting for the sake of "multi-agent" form.
- The boundaries of the roles are unclear and the outputs cover each other.
- No conflict resolution and final decision-making mechanism.
- Costs and delays are uncontrollable.

## Recommended learning tasks

- Find the Agent role definition in the project.
- Record input and output between characters.
- Analyze who is responsible for the final decision.
- Compare the complexity of single-agent and multi-agent implementations.
