# Agent Loop

## 定义

Agent Loop 是 Agent 系统的核心执行循环，通常包含观察输入、规划或思考、调用模型、选择动作、执行工具、接收结果和判断是否终止。

## 在 AI Agent 项目中的作用

它决定 Agent 能否从一次输入推进到可验证的结果，也决定最大步数、失败恢复、工具结果回传和用户中断等行为。分析项目时应优先找到 Agent Loop 的入口和终止条件。

## 典型实现方式

- ReAct 风格的 think-act-observe 循环。
- Planner + Executor 的两阶段流程。
- Workflow 节点驱动的有向图执行。
- 事件队列或任务队列驱动的异步循环。

## 常见误区

- 只看模型调用，不看循环状态和终止条件。
- 忽略工具失败后如何进入下一轮。
- 把一次 prompt 调用误认为完整 Agent。
- 没有设置最大步数、超时和人工接管路径。

## 推荐学习任务

- 找到项目中触发 Agent Loop 的入口。
- 画出一次任务从输入到结束的流程图。
- 记录最大步数、终止条件和错误处理策略。
- 比较该项目和另一个 Agent 框架的循环差异。
