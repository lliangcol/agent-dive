# Memory

## 定义

Memory 是 Agent 在一次任务内或多次任务之间保留信息的机制，包括短期上下文、会话历史、长期偏好、外部状态和可检索知识。

## 在 AI Agent 项目中的作用

Memory 决定 Agent 能否持续学习用户偏好、保存任务进度、复用历史经验，并避免每次从零开始。在学习系统中，它还用于保存项目进度和复习问题。

## 典型实现方式

- 会话级消息历史。
- 摘要压缩后的长期记忆。
- 向量数据库或文档库。
- 明确结构化的 `progress.json`、任务状态和用户偏好。

## 常见误区

- 把所有历史都当作记忆保存。
- 不区分用户事实、任务状态和模型推测。
- 记忆没有过期、纠错和删除机制。
- 敏感信息被长期保存。

## 推荐学习任务

- 找到项目保存状态的位置。
- 区分短期记忆和长期记忆。
- 检查记忆如何被注入上下文。
- 设计一个可删除、可审计的学习进度记录。
