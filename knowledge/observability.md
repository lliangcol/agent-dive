# Observability

## 定义

Observability 是通过日志、指标、追踪、事件和调试视图理解 Agent 系统运行状态的能力。

## 在 AI Agent 项目中的作用

Agent 的执行往往包含模型调用、工具调用、检索、重试和中间状态。没有可观测性，就难以排查错误、优化成本和复盘行为。

## 典型实现方式

- 结构化日志。
- Trace 记录每次模型调用和工具调用。
- Token、延迟、成本和错误率指标。
- 会话回放和中间步骤展示。

## 常见误区

- 只记录最终输出。
- 日志中泄露 prompt、密钥或用户隐私。
- 没有关联一次任务的 trace id。
- 指标无法区分模型错误、工具错误和用户输入问题。

## 推荐学习任务

- 找到项目的日志入口。
- 记录一次任务的完整执行轨迹。
- 检查是否统计 token、延迟和错误。
- 为一个失败案例补充排查日志设计。
