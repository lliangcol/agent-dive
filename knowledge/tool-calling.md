# Tool Calling

## 定义

Tool Calling 指模型或 Agent 按结构化协议选择工具、填充参数、执行外部能力并接收执行结果的过程。

## 在 AI Agent 项目中的作用

工具调用让 Agent 从文本生成扩展到文件操作、检索、数据库、浏览器、代码执行和业务 API。它也是安全、权限和可观测性的关键边界。

## 典型实现方式

- 基于 JSON Schema 的函数调用。
- 工具注册表和统一执行器。
- 工具权限白名单与参数校验。
- 执行结果回传给模型进入下一轮推理。

## 常见误区

- 只关注工具数量，不检查权限边界。
- 参数 schema 写得太宽，导致不可控输入。
- 工具异常直接暴露给模型或用户。
- 没有审计日志和重试策略。

## 推荐学习任务

- 找到工具注册位置和执行器。
- 分析一个工具的参数 schema。
- 记录工具失败时的错误回传方式。
- 增加一个只读自定义工具并验证日志。
