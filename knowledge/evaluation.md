# Evaluation

## 定义

Evaluation 是对 Agent 任务结果、执行过程、工具调用、稳定性和安全边界进行评测的方法。

## 在 AI Agent 项目中的作用

没有评测，Agent 改动很难判断是否变好。项目精读时需要关注项目是否提供样例任务、自动测试、人工评分或回归数据集。

## 典型实现方式

- 固定任务集和期望输出。
- LLM-as-judge 辅助评分。
- 工具调用轨迹检查。
- 成本、延迟、成功率和错误率统计。
- 人工 Review 与回归测试结合。

## 常见误区

- 只看单次 Demo 成功。
- 只评最终文本，不评执行轨迹。
- 使用模型评分但没有人工校准。
- 没有记录版本、配置和数据集。

## 推荐学习任务

- 找到项目的测试或评测入口。
- 设计 3 个最小任务评测用例。
- 记录成功率、成本和失败类型。
- 检查工具调用是否符合预期。
