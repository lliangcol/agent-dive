# Security

## 定义

AI Agent 安全关注工具权限、数据访问、提示注入、代码执行、文件操作、外部请求和人类审批边界。

## 在 AI Agent 项目中的作用

Agent 能调用工具后，风险从“生成错误文本”扩展到“执行错误动作”。安全边界决定项目能否用于真实工程或生产环境。

## 典型实现方式

- 工具最小权限和白名单。
- 沙箱执行代码和文件操作。
- 敏感信息过滤和脱敏日志。
- 高风险动作人工审批。
- 输入输出校验和提示注入防护。

## 常见误区

- 默认信任模型选择的动作。
- 工具权限过宽。
- 日志中记录密钥或隐私数据。
- 忽略第三方内容中的提示注入。

## 推荐学习任务

- 列出项目中所有高风险工具。
- 检查工具参数校验和权限控制。
- 设计一个提示注入测试用例。
- 记录需要人工审批的动作清单。
