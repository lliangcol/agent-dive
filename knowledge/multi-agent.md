# Multi-Agent

## 定义

Multi-Agent 指多个 Agent 以分工、协作、辩论、审查或流水线方式共同完成任务的系统形态。

## 在 AI Agent 项目中的作用

多 Agent 可以把复杂任务拆成规划、执行、审查、测试、总结等角色，但也会引入成本、延迟、冲突和状态一致性问题。

## 典型实现方式

- Supervisor 分配任务给多个 Worker。
- Planner、Executor、Reviewer 分层协作。
- 多 Agent 对话或辩论。
- 基于 workflow 图的角色节点执行。

## 常见误区

- 为了形式上“多 Agent”而过度拆分。
- 角色边界不清，输出互相覆盖。
- 没有冲突解决和最终决策机制。
- 成本和延迟不可控。

## 推荐学习任务

- 找到项目中的 Agent 角色定义。
- 记录角色之间的输入输出。
- 分析最终决策由谁负责。
- 比较单 Agent 和多 Agent 实现的复杂度。
