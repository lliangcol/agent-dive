# Security Policy

AgentDive 主要包含 AI Agent 项目学习资料、模板、图解和轻量校验脚本。即便当前不是运行时服务，公开仓库仍需要避免泄露密钥、私有路径和未经授权的第三方内容。

## 支持范围

安全报告适用于：

- 仓库中误提交的密钥、token、账号、私有路径或内部配置。
- 校验脚本、模板或文档流程中可能诱导用户泄露敏感信息的问题。
- 第三方项目收录材料中违反许可证、错误归属或包含未授权内容的问题。
- GitHub Actions、Issue/PR 模板或自动化配置中的权限与供应链风险。

不适用于：

- 被收录第三方项目本身的漏洞。请优先报告给对应上游项目。
- 没有影响路径的通用 AI Agent 安全讨论。
- 无法定位到具体文件、流程或贡献路径的抽象风险。

## 报告方式

如果 GitHub 仓库已启用 private vulnerability reporting，请优先使用该入口。否则请创建一个不公开敏感细节的 Issue，说明你需要私下报告安全问题，或联系维护者公开资料中列出的安全联系人。

报告中请包含：

- 受影响文件、目录、模板或流程。
- 复现步骤或具体证据。
- 潜在影响范围。
- 建议修复方向。

不要在公开 Issue 中粘贴真实密钥、token、账号、私有 URL、内部路径或可被滥用的完整利用细节。

## 维护者响应

维护者会优先确认报告是否影响本仓库，必要时下架相关内容、撤销暴露凭据、修正文档或调整自动化权限。修复完成后，应在 PR 或发布说明中记录影响范围和验证方式，但不公开敏感材料。
