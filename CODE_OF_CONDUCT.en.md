# Code of Conduct

AgentDive welcomes learners, maintainers, and contributors from all backgrounds. All interactions should be centered around facts, evidence, and project improvements.

## Desired behavior

- Use respectful, clear, and verifiable expressions.
- Provide sources, operational evidence, or clear human judgment boundaries for project conclusions.
- Accept contributors of varying experience levels into the discussion and help each other understand the context.
- Prioritize reproducible paths, impact scope and improvement suggestions when errors are discovered.

## Unacceptable Behavior

- Personal attacks, harassment, discriminatory language or persistent interruption of discussions.
- Falsify running results, source code conclusions, project status or contribution records.
- Submit accounts, keys, local private paths, unauthorized materials, or content that violates third-party licenses.
- Use Issues, PRs or discussion forums to post spam, advertising or irrelevant content.

## Execution method

Maintainers may edit, hide, or delete content that violates these Guidelines, and may temporarily or permanently restrict relevant participants from continuing to interact. When dealing with issues, try to retain factual evidence and bring the discussion back to project quality and community safety.

## Report

To report a security issue, please click [SECURITY.md](SECURITY.en.md). Other code of conduct issues can be reported through GitHub Issues or the maintainer's public contact information; please provide the location, time, relevant links and necessary screenshots.
