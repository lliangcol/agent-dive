# <Project Name> 图解清单

## 使用场景

用于规划、生成和检查项目图解。图解应帮助读者理解架构、流程、调用关系、集成边界和排查路径。

## 填写说明

- 每张图只表达一个核心主题。
- Mermaid 源文件优先保存为 `.mmd`。
- 导出图建议保存为 `.svg` 或 `.png`。
- 单项目图解保存到 `projects/<category>/<owner__repo>/assets/diagrams/`。

## 标准结构

### 1. 架构图

- 文件：`architecture.mmd`
- 目标：说明系统模块、模块职责和主要依赖。
- 检查：模块命名是否与源码或文档一致。

### 2. 流程图

- 文件：`flow.mmd`
- 目标：说明从用户输入到最终输出的主流程。
- 检查：箭头方向是否符合真实执行顺序。

### 3. 时序图

- 文件：`sequence.mmd`
- 目标：说明用户、Agent、模型、工具和外部服务之间的交互顺序。
- 检查：同步、异步、重试和错误分支是否清楚。

### 4. 模块关系图

- 文件：`module-relations.mmd`
- 目标：说明关键模块之间的依赖和边界。
- 检查：是否避免把目录结构简单画成架构结论。

### 5. 原理图

- 文件：`principle.mmd`
- 目标：说明 Agent Loop、Tool Calling、RAG、Memory 或 MCP 等核心机制。
- 检查：抽象概念是否与项目实现一致。

### 6. 集成图

- 文件：`integration.mmd`
- 目标：说明项目如何接入本地或业务系统。
- 检查：输入输出、权限、日志和失败路径是否覆盖。

### 7. 问题排查图

- 文件：`troubleshooting.mmd`
- 目标：说明常见问题的定位路径。
- 检查：排查步骤是否可执行。

## 图解质量标准

- [ ] 每张图只有一个主题。
- [ ] 模块名称与正文、源码或官方文档一致。
- [ ] 箭头方向准确。
- [ ] 推测内容已标注。
- [ ] 图解与正文不矛盾。
- [ ] 图下有说明和依据。
- [ ] 同一项目图解风格一致。
- [ ] 能用于 README、文档站或技术分享。
